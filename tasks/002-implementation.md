# Задача на реализацию Pineapple Pi 2.0

## Описание

Реализация многостраничного веб-сайта Pineapple Pi 2.0 на Next.js 16 (App Router) с каталогом товаров, корзиной, избранным и оформлением заказа.

## Ссылки на документацию

- 📋 [Архитектура](../docs/plans/001-architecture.md)
- 🧩 [Компоненты](../docs/plans/001-components.md)
- 🔄 [Поток данных](../docs/plans/001-data-flow.md)

---

## Этапы реализации

### Этап 1: Настройка проекта и зависимостей

**Цель:** Подготовить проект к разработке

**Задачи:**
- [ ] Установить зависимости: `gray-matter`, `marked`, `@chakra-ui/icons`
- [ ] Установить dev-зависимости: `@types/gray-matter`, `@types/marked`
- [ ] Настроить кастомную тему Chakra UI (`src/theme.ts`)
- [ ] Интегрировать ChakraProvider в root layout
- [ ] Создать структуру папок: `components/`, `lib/`, `stores/`, `types/`

**Ожидаемый результат:** Проект готов к разработке, Chakra UI с кастомной темой teal работает

---

### Этап 2: Типы данных и утилиты

**Цель:** Определить TypeScript интерфейсы и функции парсинга

**Задачи:**
- [ ] Создать `src/types/product.ts` с интерфейсом `Product`
- [ ] Создать `src/types/cart.ts` с интерфейсами `CartItem`, `CartState`
- [ ] Создать `src/lib/products.ts` с функциями:
  - `getSlugs()` — получение списка slug
  - `getProductBySlug(slug)` — парсинг одного товара
  - `getProducts()` — получение всех товаров
  - `parseMarkdownSections(content)` — парсинг секций Markdown
  - `extractTitle(content)` — извлечение H1
  - `parsePrice(priceString)` — парсинг цены
- [ ] Обработка ошибок парсинга (fallback значения)

**Ожидаемый результат:** Функции парсинга Markdown работают, типы определены

---

### Этап 3: Zustand сторы

**Цель:** Реализовать управление состоянием корзины и избранного

**Задачи:**
- [ ] Создать `src/stores/cart.ts`:
  - `useCartStore` с persist middleware
  - Actions: `addItem`, `removeItem`, `updateQuantity`, `clearCart`, `toggleDelivery`
  - Getters: `subtotal`, `deliveryCost`, `total`, `totalCount`
- [ ] Создать `src/stores/favorites.ts`:
  - `useFavoritesStore` с persist middleware
  - Actions: `addFavorite`, `removeFavorite`, `toggleFavorite`
  - Getters: `isFavorite`, `count`
- [ ] Создать `src/stores/index.ts` для экспортов
- [ ] Протестировать сторы в изоляции

**Ожидаемый результат:** Сторы работают, данные сохраняются в localStorage

---

  - Toast уведомление

- [ ] Обновить `src/app/page.tsx`:
  - Загрузка товаров через `getProducts()`
  - Отображение ProductGrid
  - Заголовок страницы

**Ожидаемый результат:** Главная страница отображает сетку товаров, кнопки работают

- [ ] Адаптивная раскладка (2 колонки на desktop)

**Ожидаемый результат:** Статические страницы товаров генерируются при сборке

### Этап 12: API роуты

**Цель:** Реализовать моковые API эндпоинты

**Задачи:**
- [ ] Создать `src/app/api/feedback/route.ts`:
  - POST метод
  - Валидация полей
  - Имитация задержки 500ms
  - Ответ: `{ success, message }`
- [ ] Создать `src/app/api/order/route.ts`:
  - POST метод
  - Валидация: корзина не пуста
  - Имитация задержки 1000ms
  - Генерация orderId
  - Ответ: `{ success, orderId, message }`

**Ожидаемый результат:** API эндпоинты работают, валидация проходит
