🎯 Цель
Разработать многостраничный (MPA) веб-сайт на Next.js (App Router) с использованием Chakra UI для компонентов и стилизации, Zustand для управления состоянием корзины и избранного. Сайт должен представлять каталог товаров с возможностью добавления в корзину, добавления в избранное, изменения количества и оформления заказа через моковый API. Ключевые требования: адаптивность (mobile-first), поддержка тем (системная/светлая/темная) с акцентным цветом teal в светлой теме, и соответствие критериям производительности (Lighthouse).

🧩 Функциональные требования и структура

1. Макет (Layout)
   Header: Горизонтальная навигация (Главная, О компании, Контакты/Форма, Избранное), иконка корзины с счетчиком (badge) количества уникальных товаров.

Footer: Три колонки:

Вертикальная навигация (ссылки на разделы).
Контакты (email, телефон, адрес — фейковые).
Год и копирайт.
Общие компоненты: Модальное окно (Popup) для принятия Cookie (отображается один раз при первом визите, хранить флаг в localStorage).

2. Структура данных товаров
   Формат хранения спецификаций:

Путь: public/products/specification/

Формат: Markdown файлы (.md)

Каждый файл содержит:

Заголовок H1: Название товара

Секция "Specification": Маркированный список технических характеристик

Секция "Price": Цена товара (например, "$40")

Пример файла:

markdown

# Banana Pi BPI-M4 Berry with Allwinner H618, 2G RAM,8G eMMC onboard

## Specification

- Allwinner H618, Quad-core ARM Cortex™-A53 processor
- ARM Mali G31 GPU
- WIFI & Bluetooth
- 2G LPDDR4 RAM
- 8G eMMC flash memory
- 1x USB2.0 Type-C OTG, 5V power supply
- 4x USB2.0 Type-A
- 1x HDMI 2.0a
- 1x 3.5mm Audio & TVE jack socket
- 1x GbE Ethernet port

## Price

$40
Изображения:

Путь: public/products/images/

Формат: JPG/PNG

Связь с товаром: имя файла изображения должно соответствовать имени файла спецификации (например, banana-pi-bpi-m4-berry.jpg для banana-pi-bpi-m4-berry.md)

3. Страницы (Pages)
   Главная (/):

Сетка (Grid) карточек товаров.

На каждой карточке товара:

Изображение, название, краткое описание (первые 3 пункта спецификации или первые 100 символов), цена

Иконка сердечка для добавления/удаления из избранного (визуально показывает состояние: пустое/заполненное)

Кнопка «В корзину» или иконка корзины для быстрого добавления

При клике на карточку (кроме иконок) происходит переход на детальную страницу товара.

Страница избранного (/favorites):

Сетка (Grid) карточек товаров, добавленных в избранное (аналогично главной странице).

Если избранное пусто — отображается информационное сообщение с предложением перейти в каталог.

На каждой карточке:

Возможность удалить из избранного (иконка сердечка)

Кнопка «В корзину» для быстрого добавления

При клике на карточку (кроме иконок) происходит переход на детальную страницу товара.

Состояние избранного синхронизировано с главной страницей и детальными страницами.

О компании (/about):

Статический контент (фейковое описание компании по производтсву компьютеров PineapplePi).

Форма обратной связи (/contact):

Поля: Имя, Email, Сообщение (использовать компоненты Chakra UI).

Интеграция с моковым REST API (например, POST /api/feedback). При успешной отправке — уведомление.

Детальная страница товара (/product/[id]):

Генерация статических страниц (SSG) для товаров на основе Markdown файлов.

Содержит:

Полное название (из H1)

Изображение (из public/products/images/[id].jpg/png)

Иконку сердечка для добавления/удаления из избранного (с состоянием)

Полная спецификация (все пункты из секции Specification, отформатированные в список)

Цена (из секции Price)

Поле ввода количества (NumberInput из Chakra UI)

Кнопку «В корзину»

Важно: Должна быть возможность положить товар в корзину по указанной цене с выбранным количеством.

Страница корзины (/cart):

Список добавленных товаров (данные хранятся в Zustand).

Возможность изменить количество (инкремент/декремент/ввод вручную) или удалить товар.

Доставка (опционально):

Чекбокс или переключатель "Добавить доставку" с ценой $5

При выборе доставки стоимость добавляется к итоговой сумме

Доставка применяется к заказу в целом (не к каждому товару)

Состояние выбора доставки хранится в Zustand store

Итоговая сумма с разбивкой:

Сумма товаров

Доставка (если выбрана) — $5

Итого к оплате

Кнопка «Оформить заказ»: отправка данных корзины и выбора доставки на моковый REST API (например, POST /api/order), очистка корзины после успешного ответа.

Not Found (/not-found или app/not-found.tsx):

Кастомная страница 404.

4. Управление данными
   Парсинг Markdown файлов:

Использовать библиотеку для парсинга Markdown (решение принимает архитектор).

Стратегия загрузки:

На этапе сборки (build time) читать все .md файлы из public/products/specification/

Парсить каждый файл для извлечения: названия (H1), спецификации (список), цены

Генерировать массив объектов товаров

Использовать это для SSG страниц товаров и главной страницы

Модель данных товара:

Архитектор самостоятельно определяет структуру объекта товара (интерфейс/тип), исходя из структуры Markdown файлов и требований к отображению.

Должны быть предусмотрены: идентификатор (slug), название, полная спецификация (массив характеристик), цена (числовое значение и форматированное для отображения), путь к изображению, краткое описание для карточки товара.

Состояние (Zustand):

Архитектор проектирует два независимых стора (или один объединенный) с учетом:

Стор корзины:

Хранение списка товаров с количеством

Хранение состояния выбора доставки (boolean)

Методы: добавление, удаление, изменение количества, очистка корзины, переключение доставки

Вычисляемые значения: сумма товаров, стоимость доставки (0 или $5), итоговая сумма, общее количество товаров

Персистентность (сохранение корзины и выбора доставки между сессиями)

Стор избранного:

Хранение списка идентификаторов избранных товаров (массив ID/slug)

Методы: добавить в избранное, удалить из избранного, проверить наличие товара в избранном

Вычисляемые значения: количество избранных товаров (для отображения в навигации, опционально)

Персистентность (сохранение избранного между сессиями)

Архитектор самостоятельно определяет оптимальную структуру сторов, методы и геттеры, а также решает: объединять в один стор или разделять.

Темизация (Chakra UI):

Использовать встроенную систему тем Chakra UI.

Настроить кастомную тему для соответствия требованиям:

Акцентный цвет в светлой теме — teal (цветовая схема Chakra)

Цвет для навигации и кнопок должен соответствовать цветовой схеме teal

Хранение выбранной темы в localStorage (автоматически обрабатывается Chakra UI)

5. UI/UX и стилизация
   Адаптивность: Mobile-first подход с использованием пропсов base, md, lg в компонентах Chakra UI. На мобильных устройствах сетка товаров (1-2 колонки), на десктопе — 3-4 колонки.

Темизация: Полностью на базе Chakra UI ColorMode. Все компоненты должны корректно отображаться в обоих режимах.

Cookie-баннер: Кастомный модальный компонент на основе Chakra UI.

Избранное:

Иконка сердечка с плавной анимацией при клике (заполнение/опустошение)

Акцентный цвет для активного состояния (заполненное сердце) — красный или teal (на усмотрение архитектора, но должно контрастировать с акцентным цветом темы)

При добавлении/удалении из избранного — визуальная обратная связь (toast уведомление или анимация)

В навигации может отображаться счетчик избранных товаров (опционально)

Доставка в корзине:

Компонент выбора доставки с чекбоксом и пояснением

Визуальное выделение при выборе доставки

Прозрачное отображение влияния на итоговую сумму

6. Технические особенности
   Работа с файловой системой:

Все данные должны быть доступны статически (SSG), без клиентских запросов к API для получения товаров

Изображения оптимизировать через next/image

Моковые API эндпоинты:

Архитектор определяет структуру эндпоинтов и формат запросов/ответов

POST /api/feedback — обработка формы обратной связи

POST /api/order — оформление заказа с передачей информации о товарах, количестве, выборе доставки и сумме

Синхронизация состояния:

Состояние избранного должно быть доступно и синхронизировано на всех страницах (главная, детальная, избранное)

При добавлении/удалении из избранного на одной странице, состояние обновляется во всех компонентах, где отображается иконка сердечка
